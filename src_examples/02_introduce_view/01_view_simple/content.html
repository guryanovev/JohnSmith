This example demonstrates simple view rendering.

Note that a View requires a template to render. Template is a selector that points to a hidden markup block. Usually such blocks defined with script element: 

<script id="TEMPLATE_ID" type="text/view">
    MARKUP GOES HERE
</script>

Optionally a View could have an init method. It is a point where you can define bindings, nested views and so on. Note that you use this.bind instead of js.bind inside a view. This makes sure that all bindings are configured relatively to the View Context. It means that if you render view to #myView and bind to .firstName then actual binding will be setup to #myVew .firstName.

Once View class is defined you can create a view instance by calling js.createView method. That method returns special View instance that could be rendered with renderTo method.
